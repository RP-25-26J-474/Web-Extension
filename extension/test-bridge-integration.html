🔗 AURA Bridge Integration – Test Page

Use this page to test ping-pong and token broadcast. Open from localhost so the extension content script runs: http://localhost:3000/extension/test-bridge-integration.html (with API server) or npx serve extension -p 5000 then http://localhost:5000/test-bridge-integration.html.

1. Extension presence (AURA_EXT_PING → AURA_EXT_PONG)

Sends AURA_EXT_PING, expects AURA_EXT_PONG with extensionPresent, loggedIn, token, user.


    


    
    

      
2. Token-only (AURA_EXT_TOKEN_PING → AURA_EXT_TOKEN_PONG)

      
Sends AURA_EXT_TOKEN_PING, expects AURA_EXT_TOKEN_PONG with token. Note: The displayed token is masked (***). Do NOT copy it – the full token is stored for the API test below.

      
      

      

    


    
    

      
3. Listen for AURA_USER_UPDATE

      
Log in or register in the extension popup. Broadcast fires after login, or after onboarding completes for registration. Shows isRegistration: true or isLogin: true.

      
      

      

    


    
    

      
4. Aggregated batches API

      
Fetches 10-second aggregated interaction windows (clicks, scrolls, etc.) saved by the extension. Integration endpoint: GET /api/interactions/aggregated-batches/last-24h (last 24h). Returns { batches, count }.

      
Leave the field empty – token from ping above is used automatically. Or paste the full JWT (starts with eyJ). Do NOT paste the masked ***... value.

      
      
      

      
      
      

      

    


    
    

      
5. Impairment profile API

      
Fetches the user's impairment profile (vision, motor, literacy scores). Endpoint: GET /api/onboarding/impairment-profile. Requires token from ping above.

      
      

      

    


    
    

      
6. ML profile ping-pongs

      
Personalized: fetched from impairment API when onboarding completes (or daily GET API for login). If "not available" after onboarding: 1) Start dummy server: cd dummy-ml-profile && npm start, 2) Click "Retry: fetch initial profile" below.

      
      
      
      
      
      

      

    


    
    

      
7. Run all tests

      
      
Runs extension ping and token ping. Log in via extension for best results.

    


    
    

      
8. Dummy ML Profile API

      
Test the dummy ML server directly. Start it first: cd dummy-ml-profile && npm install && npm start (runs on port 4000). Configure extension/config.module.js with ML_PROFILE_API_URL and IMPAIRMENT_TO_ML_PROFILE_API_URL pointing to http://localhost:4000.
